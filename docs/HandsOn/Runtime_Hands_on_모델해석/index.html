4. 모델해석 모델 해석

모델 해석

<튜토리얼 목표>

  • 모델 해석이란 모델이 예측한 결과에 대해 왜 그렇게 예측했는지 설명력을 제공합니다.
  • 본 튜토리얼에서는 생성된 모델을 이용하여 해석을 수행해 보고 결과를 확인할 수 있습니다.
  • 본 튜토리얼에서는 iris 모델을 기준으로 예시를 제공합니다.
  • 사전 조건: 모델을 생성하고, 런타임으로 배포가 완료되어야 합니다.

1. 사전 준비​

  • 자세한 내용은 해당 튜토리얼을 참고하십시오.
  1. 테스트용 모델 생성
    [AccuInsight+] 프로젝트 목록 > 프로젝트 선택 > 워크플로우 생성 > AutoML로 바로가기 선택
    <예시> 데이터셋 : Amazon S3 > accutuningawsbucket > iris.csv 선택
  2. 모델 레지스트리 등록
    [AccuInsight+]
    프로젝트 목록 > 프로젝트 선택 > 워크플로우 선택 > 모델 노드 선택 > 모델 레지스트리 등록 > 새 모델로 등록 선택
  3. 모델 배포 및 서빙
    [AI Runtime]
    프로젝트 명 / 모델 레지스트리 > 모델 선택 > 서빙 시작 선택
    <예시> 자원설정 = Small 선택 -> 서빙 시작 선택

2. 모델 선택​

  • 모델 목록에서 모델을 클릭하여 들어가서 [해석 이력] 탭을 클릭합니다.
  • [해석 이력]화면으로 들어가면 우측 상단에 [모델 해석]이라는 버튼이 있습니다. [모델 해석]버튼을 눌러서 다음 화면(자원 선택)으로 이동합니다.

3. 자원 선택 및 데이터 입력​

  • 해석 모델이 동작할 수 있는 정도의 자원을 선택하고, 화면 하단에 테스트 데이터를 입력한 후 해석 요청 버튼을 누르면 해석 요청이 완료된다.
    <예시> Small 선택: 큰 자원을 선택 시 자원 부족으로 정상적인 수행이 어려울 수 있으므로 실습 시에는 Small을 선택하도록 합니다.

<예시>
데이터 타입 = Tabular
Tabular 데이터 등록: [+Add] 클릭 > [4.7, 3.2, 1.3, 0.2] 입력 (반드시, []으로 묶어야 합니다.)

  • 입력이 끝나면 [해석 요청] 버튼을 클릭합니다.

  • 데이터 구조 및 형식

    Tabular 모델의 데이터는 리스트 형태로 학습에 들어간 컬럼의 개수만큼 데이터를 입력해야 합니다. 예) [5.1, 3.5, 1.4, 0.2]

4. 해석 요청​

  • 해석 요청이 되면 모델이 기동되기까지 잠시 시간이 소요된 후 해석 이력 탭으로 화면이 이동됩니다.
  • 해석이 완료될 때까지 보통 (가벼운 모델일 경우) 건당 1~2분의 시간이 소요되며, 해석이 완료되면 해당 레코드 별로 결과 확인이 가능합니다.

5. 해석 결과 조회​

  • 해석 이력 버튼을 클릭하면 기본으로 현재 시간 기준으로 3시간 이전까지 이력이 조회됩니다.
  • 조회 대상 시간은 아래와 같이 최근 3시간(기본), 최근 2시간, 최근 1시간, 직접 설정 등으로 지정할 수 있으며, 직접 설정 시 임의의 날짜와 시간을 사용자가 설정할 수 있습니다.

  • 이력을 조회하면 해당 시간 동안 발생한 이력에 대해 [이력 번호, 요청 시간, 소요 시간, ID(요청자), 버전(모델 버전), 결과] 항목을 목록 형식으로 보여줍니다.

  • 목록에서 각각의 레코드를 클릭하면 아래와 같이 상세 결과가 팝업됩니다.
  • 상세 조회 시 하나의 해석 요청 건에 대해 결과를 확인할 수 있으며, anchor 정보를 통해 예측 결과에 대한 예측 근거를 확인할 수 있습니다.

3.Drift 모니터링

드리프트 모니터링 등록 및 조회

<튜토리얼 목표>

  • 배포된 서비스에 드리프트 모니터링을 등록하여 데이터의 추세를 모니터링 합니다..
  • 모니터링 주기를 스케쥴링하고 데이터에 이상이 감지되면 NOTI를 수신할 수 있습니다.

1. 모니터링 등록​

  • 서비스 중인 모델 화면으로 이동합니다.

  • “모델 Drift” 탭을 클릭하여 드리프트 목록화면으로 이동합니다.

  • “모니터링 등록” 버튼을 클릭합니다.

* 모니터링 설명 : 등록할 모니터링에 대한 설명을 작성합니다.
* 기준시각 : 분석을 시작할 시각을 입력합니다. 현재 이후의 시각이어야 합니다
* 알림 설정 : 데이터에 이상이 감지되었을 때, NOTI를 입력받을 슬랙 채널을 설정합니다.
* 분석주기 : 분석주기를 설정합니다. 기본방식과 Cron 표현식 두 가지 방법으로 설정할 수 있습니다.
    * 기본방식: 기준시각으로부터 일/주/월 단위로 분석주기를 설정합니다.
    * Cron 표현식: 가이드에 따라 Cron 형식으로 분석주기를 설정합니다.
* 기준데이터 : 두 데이터 셋을 비교함으로써 데이터 분석이 이루어집니다. 입력된 데이터 셋과 비교할 데이터 셋을 결정합니다.
    * 과거데이터 : 분석주기에 따른 이전 데이터 셋을 비교분석에 사용합니다.
    * 학습데이터 : 모델 학습에 사용했던 데이터 셋을 사용합니다.
* 대상피쳐 설정 :
    * 타입 명세: 각 피쳐의 데이터 타입이 수치형인지 범주형인지 명세합니다.
    * 피쳐 선택: 해당 모니터링에서 분석할 피쳐를 on/off로 선택합니다.
    * 임계값 설정: 각 피쳐의 변화가 얼마나 크면 “이상 데이터”로 판단할지 임계값을 설정합니다. 일괄설정할 수도 있습니다.
  • “등록하기” 버튼을 클릭합니다.

2. 모니터링 조회​

  • 등록 후 Drift 모니터링 목록 화면으로 이동합니다.
  • Drift 분석 모니터링을 활성/비활성화 할 수 있습니다.
  • 가장 최근의 분석 결과, 이상감지 되었는지를 “드리프트 여부” 에서 확인할 수 있습니다.

  • 모니터링을 선택하여 “드리프트 결과” 화면으로 이동합니다.

* 모니터링 요약
    * 모니터링 상태 : 해당 모니터링이 활성/비활성 상태인지를 확인합니다.
    * 최근 실행 일시 : 마지막 분석 일시를 확인합니다.
    * 다음 실행 일시 : 다음 분석이 이루어질 시각을 확인합니다.
    * 드리프트 여부 : 최근 실행에서 드리프트가 감지 되었는지 여부를 확인합니다.
* 상위 드리프트 피쳐 : 마지막 분석에서 가장 높은 드리프트를 보인 상위 3개의 피쳐를 확인합니다.
* 드리프트 추이: 피쳐의 드리프트 추이를 그래프로 확인합니다.
* 드리프트 이력: 드리프트 분석 이력과 Drift 스코어를 확인합니다.

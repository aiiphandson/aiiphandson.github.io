6. Hands-on No Code Modeling

No Code Modeling

<튜토리얼 목표>

  • 프로젝트와 워크플로우를 신규로 생성하고, No Coding Tool인 Wrangler 와AutoML등을 이용하여 손쉽게 데이터 전처리와 모델 개발을 수행합니다.
  1. Wrangler : 엑셀과 같은 Spread Sheet 기반의 전처리 Tool 로써 손쉽게 데이터 가공 처리를 하기 위함
  2. AutoML : 통계적 지식이나 컴퓨터 프로그래밍 지식이 없는 사람도 손쉽게 머신러닝 모델을 개발하기 위한 목적으로 모델개발 단계 중 일부 또는 전체를 자동화하는 기술을 의미함
  • AutoML학습 모델을 New 데이터에 적용하여 예측값 생성을 목표로 합니다.

1. 프로젝트 생성​

  • 먼저 사번으로 로그인을 합니다. 초기 비밀번호는 P@s$w0rd입니다. 처음 로그인하면 새 비밀번호를 설정해야 합니다.

새 비밀번호를 입력 후 새로운 비밀번호로 로그인 하세요

  • 대시보드 > 프로젝트에서 [+프로젝트] 버튼을 클릭하여 프로젝트를 신규로 생성합니다. 프로젝트는 팀당 하나씩만 생성해 주세요

  • 대시보드 > 프로젝트 등록에서 프로젝트 필수 정보를 입력하고, 자원 선택 후 [등록하기] 버튼 클릭합니다.
이미지 변경 : 원하는 이미지를 선택하세요
프로젝트 제목(필수) : 프로젝트 제목을 입력하세요
프로젝트 내용 (필수) : 프로젝트의 내용을 작성하세요
프로젝트 사용자 등록 : 프로젝트의 팀원 선택 후 오른쪽 이동 버튼을 클릭하여 추가 후 권한을 Owner로 변경해주세요
Livy 선택 : 사번이홀수로 끝나면 livy_contest_01,짝수로 끝나면 livy_contest_02를 선택하세요
클러스터 선택 : Default로 놔두세요
자원 설정 (필수) : Medium(8Core 16G)를 선택하세요

2. 워크플로우 생성​

  • [+워크플로우] 버튼을 클릭하여 워크플로우를 신규로 생성합니다.

  • 워크플로우 작업선택 에서 [빈 워크플로우] 를 선택 후 우측하단의 [다음] 버튼 클릭합니다.

  • 워크플로우 생성 > 데이터셋 추가 에서 [Asset] 버튼을 클릭하고, 이후 나타나는 Pop-up 창에서 데이타를 선택하고 [등록]버튼을 클릭합니다.
    <예시> 데이터 명 : churn_train

  • 데이터셋 추가 화면으로 돌아와서 우측 상단의 [추가] 버튼을 클릭합니다.

3. 데이터 전처리​

  • (전처리 예시 1: Record 중복값 처리) 워크플로우에 생성된 [데이터셋] 노드를 클릭하고 우측패널에서 [전처리] 버튼을 클릭합니다.

  • 전처리 메뉴 중 “기본처리” > “Drop Duplicate” 를 차례로 클릭합니다.

  • 우측 패널의 Drop Duplicate 설정화면에서 전체 Record 중복값을 처리하기 위해서 [+추가] 버튼을 클릭합니다.
    • 참고) 특정 컬럼의 중복값을 처리할 경우는 [더 보기] 버튼을 클릭하여 해당 컬럼명을 지정 해 줍니다.

  • (전처리 예시 2: Null 값 처리) [Drop Duplicate] 노드 우측에 연결된 [데이터 셋] 노드를 클릭하고 우측 패널에서 [전처리] 버튼을 클릭합니다.

  • 우측 패널 전처리 메뉴 중에서 [Wrangler] 버튼을 클릭합니다.
    • 참고) 전처리 작업은 개별 노드 (기본처리 노드, 고급처리 노드 등) 에서 처리할 수도 있지만 Wrangler 를 이용하여 하나의 화면에서도 많은 전처리 작업이 가능합니다.[Wrangler 기능]
      1. 셀 단위 전처리 : 공백제거, 찾아바꾸기, 누락채우기, 문자열 첨가
      2. 열 단위 전처리 : 데이터타입 변경하기, Sorting, 열 나누기, 문자열 추출하기, New Column 생성하기, Keep, Delete
      3. 행 단위 전처리 : 값에 따른 필터링, 상위 N% 필터링, Limit, 열 중복제거, 전체 중복 제거, 열 결측치 제거
      4. 기타 전처리 : Time Stamp 추가, 날짜 포맷 변경, Time Move

  • ‘Wrangler 그리드’ 화면에서 조회된 데이터를 확인하고, 컬럼명을 클릭한 후 컬럼명에서 마우스 우측버튼을 클릭하면 나타나는 메뉴창에서 전처리 항목을 선택합니다.

  • 특정 컬럼에서 이미 알려진 Null 값을 대체하기 위해서는 전처리 메뉴창에서 ‘누락채우기’ > ‘FillNa by col’ 항목을 클릭합니다.
    < 예시 > Balance 컬럼 Null 값 처리

  • 이후 나타나는 우측 패널에서 null 값 대신 대체할 값을 입력하고 [적용] 버튼을 클릭합니다.
    < 예시 > Null 값 -> 0 으로 대체

  • (전처리 예시 3: New Feature 생성) 2개 컬럼을 연산해서 새로운 컬럼을 생성하기 위해서는 전처리 메뉴창에서 ‘withColumn’ 항목을 클릭합니다.
    < 예시 > BaseYear 컬럼과 JoinYear 컬럼을 연산해서 새로운 Tenure (가입기간) 컬럼을 생성

  • 이후 나타나는 우측 패널의 selectType 에서 column 을 선택합니다. 그리고 연산하고자 하는 2개의 컬럼명과 연산자를 선택하고 새로운 컬럼명을 입력한 후 [적용] 버튼을 클릭합니다.
    < 예시 > col1 : BaseYear, 연산자 : (-), col2 : JoinYear, New Column 명 : Tenure

  • 전처리 작업이 모두 완료되면 우측상단의 [완료] 버튼을 클릭합니다.

4. 데이터 탐색​

  • 워크플로우에서 탐색하고자 하는 [데이터 셋] 노드를 더블 클릭합니다.

  • 데이터 조회 화면에서 “Correlation” Tab 을 클릭하여 상관분석을 진행할 수 있습니다.

  • “Interactions” Tab 을 클릭하여 변수별 분포 및 변수간 관계 탐색을 진행할 수 있습니다. 그리고 탐색 작업이 모두 완료되면 왼쪽 상단의 ‘<’ 표시 부분을 클릭하여 Dataset 화면을 빠져나옵니다.
    <예시> Y 변수와 X 변수간의 관계 분석시 설정 - 그래프 타입 : Column, X축 : Exited, Y 축 : CreditScore, Calculate : Mean
    • 참고) 그래프 타입 :Line, Column, Bar, Pie, Scatter, Box Plot

  • [데이터 셋] 노드의 차트 생성 메뉴를 이용하여 데이터 탐색을 진행할 수도 있습니다. 탐색하고자 하는 [데이터 셋] 노드에서 우측버튼을 클릭하고 “차트 생성” 메뉴를 클릭합니다.
    • 참고) 차트 생성 메뉴를 이용한 데이터 탐색 시 데이타 조회 화면에서의 탐색기능 보다 다양한 종류의 차트를 이용할 수 있고, 차트 저장 및 리포트 생성 기능이 있어 다른 사용자와 공유가 가능합니다.

  • 차트 생성 화면에서 X축 표시 왼쪽의 버튼을 클릭하고, 차트 선택영역이 나타나면 차트 종류종류 중 히스토그램 차트를 선택합니다.
    참고) 차트 종류 : Line, Area, Bar, Column, Mixed, Cumulative Column, Pie, Doughnut, Radar, Gauge, Heat Map, Histogram, Box Plot, Table, Map /Geographical, Word Cloud .

  • 오른쪽 패널 차트 설정에서 분석 대상 컬럼을 선택 후 왼쪽의 컬럼 (기준축) 란으로 드래그합니다.
    < 예시 > Age 선택

  • 구간값 설정을 통해 변수 구간 크기를 조정합니다.
    < 예시 > 구간값 : 5

  • 왼쪽 상단위 Untitled 위에 마우스 오버하면 표시되는 [연필 아이콘] 을 클릭하여 차트 이름을 변경 후 V 아이콘을 클릭, 저장 버튼을 클릭합니다.
    < 예시 > 차트명 : Age

  • 차트를 추가할 경우에는 왼쪽 하단 차트명 옆에 있는 + 버튼을 클릭하여 신규 차트를 생성하거나, 데이터 탐색이 모두 완료되면 왼쪽 상단의 ‘<’ 표시 부분을 클릭하여 차트 생성 화면을 빠져나옵니다.

5. AutoML 학습 > 설정 및 시작​

  • *참고) AutoML기능
  1. Data Preview : 데이터 미리보기
  2. Column 확인 : 컬럼명, 데이터 타입, 결측값 비율, 컬럼 분포 (분포 유형, 최빈값, 고유값, 평균값, 최소값, 최대값)
  3. AutoML 설정
    a. 기본 설정 : 분석 기법 (분류/회귀), 타깃, 지표 선택
    b. 컬럼 단위 설정 : 컬럼 사용 여부, 데이터 타입 변경, 결측치 대체, 이상치 제거, 데이터 변형 방법 선택
    c. 데이터 분할 : Holdout (Train/ Valid/ Test 비율 선택), CV (Fold 개수, Test 비율 선택)
    d. 샘플링 : 샘플링 대상 Dataset 선택 (전체 데이터 or Train 데이터), Sample Size 입력, Balancing 기능 선택 (Oversampling / Undersampling)
    e. 고급 설정 :
    1. Timeout 설정 (전체 Timeout 설정, 알고리즘 단위의 Timeout 설정), 앙상블 모델 사용여부
    2. Estimator 선택 (사용할 ML 알고리즘 선택)
    3. 범주형 변수의 경우 One Hot Encoding 선택 (미선택시 Lebel Encoding 적용)
    4. 수치형 변수의 경우 Scaling 방법 선택 (Standardize, MinMax, Normalize)
    5. Feature Engineering : Feature 선택 (SelectPercentile, SelectKBest, SelectFwe, Boruta), Feature 차원 축소 (PCA, FeatureAgglomeration), Feature 증강 (Polynomial)
  4. EDA : Correalation Matrix, Pair Plot, Variable Analysis, Parallel Coordinates
  5. Leader Board : 모델명, 모델 순위, 모델 성능 (Metric), 모델 정보 (Pipeline), 모델 차트 (Feature Importance, Confusion Matrix, AUC, Lift 등) 선택, 최종 모델 선정 (배포)
  • AutoML 학습을 위해서 최종 [데이터 셋] 노드를 클릭하고, 우측 패널에서 [모델링] 버튼을 클릭합니다.

  • 우측 패널 모델링 종류에서 “AutoML” 을 선택합니다.

  • 우측 패널에서 [+추가] 버튼을 클릭합니다.

  • [AutoML] 노드를 더블 클릭하거나, 마우스 우측 버튼 클릭 후 “AutoML 설정하기” 를 선택합니다.

  • AutoML 화면에서 번 기본설정에 셋팅된 분석기법, 타깃, 지표를 확인하고 필요시 Default 설정값을 변경합니다. <예시> 분석기법: 분류, 타깃: Exited, 지표: F1
    참고) AutoML 시작시 타깃변수는 맨 마지막에 위치한 컬럼이 자동으로 선택되고, 그 컬럼의 테이타 타입에 의해 분석기법이 자동 셋팅됩니다.

  • 왼쪽 상단의 Data Columns Tab 을 클릭하여 컬럼 정보를 확인하고, 필요시 컬럼 단위의 설정 작업을 진행합니다.
    < 예시 > 사용여부 체크박스 : CustomerId, Surname, BaseYear 컬럼 제외

  • 데이터 분할이 필요한 경우 분할 (Optional) 옵션에서 Holdout 또는 CV 방법을 선택합니다.
    <예시 > Holdout > Test 사용 > Train : Valid : Test 비율 = 6 : 2 : 2

  • 샘플링이 필요한 경우 샘플링 (Optional) 옵션에서 Sampling Target 데이터를 선택하고, Sampling Size 를 입력합니다. 그리고 Oversampling 또는 Undersampling 이 필요한 경우 [Balancer] 버튼을 클릭합니다.

  • [Balancer] 버튼을 클릭하면 나타나는 Pop-up 창에서 기준 Class의 막대 그래프를 더블 클릭하면 비교 Class 의 Size 가 기준 Class 의 Size 와 동일하게 조정됩니다. Ex) ‘0’ Class 를 더블 클릭하면 ‘1’ Class 의 Size 가 ‘0’ Class 와 동일하게 조정됨

  • 그 외 추가 설정이 필요한 경우 고급설정 옵션에 들어가서 추가 설정을 진행합니다.

  • 모든 설정이 완료되면 오른쪽 상단의 [실행] 버튼을 클릭합니다.

  • 기본 설정값을 재확인하는 Pop-up 창이 뜨면 설정값 확인 후 [실행하기] 버튼을 클릭합니다.

6. AutoML 학습 > 리더보드​

  • AutoML 실행이 완료되면 워크플로우에서 [모델 노드] 가 생성되고, [모델 노드] 를 클릭하면 우측 패널에서 모델 성능을 확인 할 수 있습니다. 그리고 상세한 모델 결과를 확인하기 위해서는 [AutoML 노드] 를 더블 클릭합니다.

  • Learderboard Tab 을 클릭하여 알고리즘 순위, 알고리즘별 모델 성능 등 상세한 모델 결과를 확인할 수 있습니다. 최종 모델을 변경하고자 할 경우에는 해당 알고리즘 우측에 있는 [배포] 버튼을 클릭하고, 모든 작업이 완료되면 왼쪽 상단에 있는 ‘<’ 표시를 클릭하여 화면을 빠져 나옵니다.

  • 워크플로우 이름을 변경하기 위해 좌측상단 워크플로우 이름 위에 마우스 오버하면 표시되는 [연필 아이콘] 을 클릭한 후 이름을 변경하고 [V] 아이콘 클릭합니다. 그리고 워크플로우를 저장하기 위해서 우측 상단의 [저장] 버튼을 클릭합니다.
    < 예시 > 워크플로우명 : AutoML학습

7. AutoML 예측​

  • [모델 노드] 를 클릭한 후 우측 패널에서 [예측용 Worlflow 생성] 버튼을 클릭합니다

  • “예측용 워크플로우 복제 제한 안내” Pop-up 창이 뜨면 [적용] 버튼을 클릭합니다.
    참고) 학습 목적으로 사용되었던 Record 처리 기능 중 Dup 처리, Sampling 기능 등은 예측 목적에서는 제외됨

  • 대시보드 > 프로젝트 화면으로 돌아가서 워크플로우 이름에 Predict.~ 이름이 붙어 있는 예측용 워크플로우가 생성된 것을 확인하고 더블 클릭합니다.
    <예시> Predict.AutoML학습

  • 예측용 워크플로우에 들어와서 [데이터 가져오기] 노드를 클릭합니다.

  • 데이터셋 추가 화면에서 [Asset] 버튼을 클릭한 후, 에셋 선택 Pop-up 창이 나타나면 예측용데이터를 선택하고 [등록] 버튼을 클릭합니다.
    < 예시 > churn_test

  • 데이터셋 추가 화면의 우측 상단의 [+추가] 버튼을 클릭합니다.

  • 워크플로우에 들어와서 [모델 노드] 를 클릭 한 후 우측 상단의 [예측] 버튼을 클릭합니다.

  • [모델 노드] 옆에 예측 [데이터 셋] 노드가 정상적으로 연결된 것을 확인한 후, 예측 [데이터 셋] 노드를 더블 클릭한 후 예측 데이터를 확인합니다.

  • 예측 데이터 셋에서 마지막 컬럼으로 이동하여 예측값 컬럼인 Predict_Value 값을 확인합니다. 그리고 우측 상단 끝에 있는 점 표시 부분을 클릭하여 데이터를 저장합니다.
    < 예시 > 다운로드를 선택하여 csv 파일로 저장

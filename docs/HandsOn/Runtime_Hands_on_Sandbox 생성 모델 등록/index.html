2.Sandbox 생성 모델 등록

Sandbox 생성 모델 등록

<튜토리얼 목표>

  • 워크플로우를 신규로 생성하고, Coding Tool인 *SandBox 을 이용하여 모델 개발을 수행합니다.
  • 모델의 배포 가능여부를 테스트하고 실제 서비스로 배포하는 과정을 수행합니다. *SandBox: 데이터 분석가들이 주로 사용하는 RStudio, Jupyter 에서 데이터 처리 및 분석 작업 할 수 있는 환경을 의미함

1. 워크플로우 생성​

  • [+워크플로우] 버튼을 클릭하여 워크플로우를 신규로 생성합니다.

  • 워크플로우 작업선택에서 [Sandbox로 바로가기] 선택 후 우측하단의 [다음] 버튼 클릭합니다.

  • Sandbox 분석환경 설정 화면에서 설정 후 우측 하단 [분석환경 생성하기] 버튼을 클릭합니다. < 예시 > 분석환경 : Jupyter Lab, 자원설정 : Small

  • Jupyter 비밀번호 설정 Pop-up 창에서 비밀번호를 입력 후 [등록] 버튼을 클릭합니다. < 예시 > 비밀번호 : abcd1234!

  • 워크플로우에서 우측 패널의 [데이터 가져오기] 를 선택합니다.

  • 데이터셋 추가 에서 [Asset] 버튼을 클릭하고, 이후 나타나는 Pop-up 창에서 학습 데이터를 선택하고 [등록]버튼을 클릭합니다.
  • 데이터 명 : iris

  • 데이터셋 추가 화면으로 돌아와서 우측 상단의 [추가] 버튼을 클릭합니다.

  • 워크플로우에서 2개의 [데이터 셋] 노드를 적당한 위치로 옮기고 [Jupyter lab] 노드와 연결합니다. 노드를 가까이대면 연결되기도 하지만 표시된 위치를 클릭하고 드래그해도 됩니다.
  • 그리고 [Jupyter lab] 노드를 클릭하면 나타나는 우측 패널에서 상태가 ‘Running’ 인지 확인하고 [접속] 버튼을 클릭합니다.

  • jupyter notebook 비밀번호를 입력하고 [Log in] 버튼을 클릭합니다. < 예시 >Password : abcd1234!

2. Sandbox 노드 사용​

  • Jupyter 화면으로 들어오면 위에 테두리로 표시된 Template을 클릭한 뒤 팝업 창에서 template-scikitlearn 을 클릭하고 Go 버튼을 누릅니다.

  • 좌측에 Untitled.ipynb 라는 파일이 생성됩니다. 생성된 파일의 코드들을 순서대로 실행하면 모델 배포까지 수행할 수 있는 가이드 템플릿입니다.

  • 데이터 가져오기 코드를 실행하면 연결한 데이터셋 노드의 데이터를 사용 준비합니다. 만약 여러개의 데이터셋 노드를 샌드박스에 직접적으로 연결했다면 첫 번째 데이터셋 노드만을 사용합니다. -아래 코드를 실행하면 데이터 가져오기를 수행하고, 가져온 데이터의 정보를 확인할 수 있습니다. 데이터는 Pandas DataFrame의 형태로 가져옵니다.

  • 배포할 Scikit-learn 모델을 직접 작성해볼 수 있습니다. model, preprocessor, postprocessor를 클릭하면 해당 ipynb 파일로 링크를 통해 이동합니다.

  • 위와 같이 Scikit-learn으로 작성된 기본 모델을 확인하실 수 있습니다. 현재는 LogisticRegression 모델이 예시로 작성되어 있습니다.

  • 모델에 사용 할 데이터를 준비합니다. 앞서 데이터 불러오기 작업에서 dataset이라는 변수를 생성했습니다. -dataset 변수에는 불러 온 데이터가 Pandas DataFrame 형태로 저장되어 있습니다. 예시로 사용한 dataset의 정보를 아래와 같이 한번 살펴보겠습니다.

  • 모델 훈련을 하기 위해 Target 데이터를 분리해줍니다. 준비된 iris 데이터셋에서의 Target column은 “Variety” 입니다. 간단하게 data_x, data_y로 훈련용 데이터를 준비합니다.

  • 이제 사용할 데이터까지 준비되었으니 모델 훈련을 수행하면 됩니다.
    준비되어 있는 코드를 순서대로 실행합니다.

  • 위와 같이 input 데이터를 모델에 전달하여 모델이 예측 데이터를 생성할 수 있는지 확인해보겠습니다.

  • 이제 모델을 배포하기 위해 저장하겠습니다. 모델은 지정한 경로에 저장되어야 하므로 아래의 코드를 실행해서 저장합니다. 또한 순차적으로 코드를 실행하여 모델이 정상적으로 저장되었는지 확인해보겠습니다.

  • Run Test Server 코드를 실행하면 아래와 같이 생성한 모델을 테스트 배포할 수 있는 메뉴가 생성됩니다.
  • Select 버튼을 눌러서 작업에 사용된 데이터인 iris.csv를 선택하고 Test Server: Start 버튼을 클릭합니다.

  • 모델의 테스트 배포가 성공하면 아래와 같이 하단 로그에 running server ‘test’ … Succeeded 메시지가 출력됩니다.

  • 이제 테스트로 배포한 모델에 예측하고자 하는 데이터를 입력하겠습니다. 모델 예측 요청 전달 부분의 코드를 실행하면 위와 같이 데이터를 배포된 모델에 입력할 수 있는 메뉴가 생성됩니다.
  • JSON 탭을 클릭하면 우측 아래와 같이 json 형태의 데이터를 입력할 수 있습니다. 입력 데이터의 예시들이 코드의 아래에 준비되어 있습니다.
  • 화면을 내리면 아래와 같이 테스트용 요청 샘플이 준비 되어있습니다.
  • 해당 셀의 내용을 그대로 복사해서 붙여넣습니다.

  • 예시의 형식처럼 데이터를 입력하고 Send버튼을 클릭하면 아래의 화면에 prediction 결과가 나오는 것을 확인할 수 있습니다.
  • 테스트 서버에 배포된 모델에 대해 예측이 성공하면 아래와 같이 Json 형식으로 prediction 으로 예측 값이 생성됩니다.

3. 모델 배포​

  • API 테스트가 끝나면 실제 모델을 배포하기 위해 저장한 모델을 추출하겠습니다. 샌드박스 화면을 나와서 워크플로우로 돌아오면 우측 탭에서 "모델 추출" 버튼을 클릭할 수 있습니다. 그러면 위와 같이 model 노드가 생성됩니다.

  • model 노드를 마우스 우클릭한 뒤 모델 레지스트리 등록을 클릭하면 모델을 배포할 수 있습니다.

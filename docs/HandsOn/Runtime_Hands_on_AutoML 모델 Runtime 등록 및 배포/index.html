1. AutoML 모델 Runtime 등록 및 배포

AutoML 모델 Runtime 등록 및 배포

<튜토리얼 목표>

  • AutoML을 통해 생성된 모델을 AI Runtime에 등록, 배포합니다.

    AutoML 이란? 통계적 지식이나 컴퓨터 프로그래밍 지식이 없는 사람이라도 손쉽게 머신러닝을 이용할 수 있도록 모델링 단계 중 일부 또는 전체를 자동화하는 기술을 의미함

    AI Runtime 이란? AccuInsight+에서 생성된 모델을 등록 관리하고 그 모델을 활용 예측 API를 노출하여 운영 환경에서 타 시스템에서 활용할 수 있도록 배포 및 운영 관리할 수 있는 기능을 제공함.

  • 배포된 모델을 통해 예측 테스트를 실행해 보고, 예측 이력을 조회해 봅니다.

  • 배포 중인 모델에 대한 추가 버전을 등록하고, Canary 배포 후 프로모션 해 봅니다.

  • 배포된 모델을 중지해 봅니다.

1. 새 모델 등록​

  • 모델 개발 워크플로우에 들어와서 [모델] 노드 우측버튼을 클릭하여 모델레지스트리 등록 > 새 모델등록을 선택합니다.

  • 모델 레지스트리 등록 Pop-Up 창에서 모델 이름을 입력하고, 중복확인 후 확인 버튼을 클릭합니다.

  • [모델 노드]를 클릭 한 후 우측 패널 모델 레지스트리 등록 이력에서 모델 등록일시를 확인하고 유저명 우측의 버튼을 클릭하여 AI Runtime 으로 이동합니다.

  • AI Runtime 화면에서 모델 상태가 배포가능 상태인지 확인합니다.

2. 배포 자원 설정​

  • 모델 버전 현황에서 [서빙시작] 버튼을 클릭하면 배포자원 없음 팝업이 보이고 팝업 창 내 [배포자원 생성] 버튼을 클릭하여 배포자원 생성 화면으로 이동합니다.

    (이미 배포자원 생성을 한 경우는 바로 팝업이 뜨지 않고 서빙 설정 화면으로 이동됩니다.)

  • 배포자원 생성 팝업창에 배포자원 설명 입력, 자원 선택 후 [생성완료] 버튼을 클릭합니다.
    <예시>
    • 배포자원 설명 입력(필수): 테스트 배포 자원
    • 자원설정(필수): Medium 선택
    • GPU 설정 추가: 선택하지 않음.

3. 모델 서빙​

  • 다시 [서빙시작] 버튼을 클릭하여 ‘서빙 설정’ 화면으로 이동하여 서빙 설정 입력 후 하단의 [서빙시작] 버튼을 클릭해 배포 요청을 완료합니다.
    <예시>
    • Concurrency: 100(default)
    • 자원설정(필수): Small 선택
    • Replica: 1~1(default)
    • GPU설정: 사용안함(default)

  • 서빙 시작 요청 후 배포가 완료될 때까지 버전현황 tab의 서빙 상태가 WAITING으로 표시되는 것을 확인할 수 있습니다.

  • 배포가 완료되면 서빙상태가 RUNNING 상태로 바뀌는 것을 확인할 수 있습니다. RUNNING 상태에서는 API로 예측 요청을 해 볼 수 있습니다.

4. 예측 테스트 및 이력 조회​

  • 예측이력 탭을 클릭하여 이동한 후 [예측 테스트] 버튼을 클릭합니다.

  • ‘예측 테스트’ pop-up 창에서 학습데이터를 활용해 자동 생성된 입력 데이터를 확인한 후 [전송] 버튼을 클릭하여 요청합니다.

  • 예측 테스트 완료 후 출력데이터에 예측 결과값이 표시되는 것을 확인합니다.

  • [확인]버튼을 클릭하여 예측 테스트 팝업을 닫고 예측이력 리스트 우측 상단의 [다시조회] 이콘 버튼을 클릭하여 테스트 한 예측이력이 추가로 조회되는 것을 확인 할 수 있습니다.

  • 조회된 이력을 클릭하면 입력, 출력 메시지를 포함한 이력 메시지 상세 정보를 볼 수 있습니다.

5. 모델 버전 등록​

  • 버전 현황의 모델 버전 추가 메뉴>’워크플로우 조회’를 클릭하여 해당 모델을 생성했던 workflow로 이동합니다.

  • [AutoML] 노드를 마우스 우측 글릭 후 AutoML 설정하기를 선택합니다.

  • AutoML leaderboard에서 배포 완료된 모델 외 다른 모델을 골라 [배포] 버튼을 클릭한 후 workflow 화면으로 돌아옵니다.

  • 배포를 요청한 모델이 workflow 상에 추가로 표시되는 것을 확인할 수 있습니다.

  • 추가된 모델을 선택하고 마우스 우측버튼 클릭하여 ‘모델레지스트리 등록> 새 버전으로 등록’을 클릭합니다

  • 버전을 등록할 모델을 선핵하고 [확인]버튼을 클릭하여 새 버전으로 모델을 등록합니다.

  • [모델 노드]를 클릭한 후 우측 패널 모델 레지스트리 등록 이력에서 모델 등록일시를 확인하고 유저명 우측의 버튼을 클릭하여 AI Runtime으로 이동합니다.

  • AI Runtime 화면에서 등록한 버전 상태가 [빌드중]으로 표시되는 것을 확인합니다.

  • 일정 시간 경과 후 추가 등록 버전의 상태가 [배포가능] 상태로 변경되는 것을 확인합니다.

6. Canary 배포​

  • 새로 등록한 버전의 [canary 배포 추가] 버튼을 클릭합니다.

  • Default 서비스와 Canary 서비스가 받는 요청 분배 비율을 원하는 값으로 설정합니다.
    <예시> 서비스 버전(Default): 90%, 서비스 버전(Canary): 10%

  • Canary 서비스 자원 및 설정 값을 선택하고 우측 하단의 [서빙 시작]버튼을 클릭합니다.

  • 버전 현황에서 Canary 배포 비율 및 상태가 표시되는 것을 확인합니다.

7. 모델 프로모션​

  • [서비스 대시보드] 탭을 클릭하여 서빙 설정을 먼저 확인한 후 ‘canary 서빙버전’의 [상태변경] selectbox에서 프로모션을 선택합니다.

  • Canary 서빙 프로모션 확인 팝업창에서 [확인] 버튼을 클릭합니다.

  • Default 서빙 버전이 Canary 서빙 버전으로 대체되었음을 볼 수 있습니다.

8. 모델 서빙 중지​

  • [서비스 대시보드] 탭에서’ Default 서빙버전’의 상태변경 select box에서 [중지]를 선택합니다.

  • Default 서빙 중지(삭제) 팝업에서 [확인] 버튼을 클릭합니다.

Default 서비스가 중지되어 ‘DELETED’ 상태로 표시되는 것을 확인할 수 있습니다.

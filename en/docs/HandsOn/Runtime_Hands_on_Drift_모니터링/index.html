3. Drift Monitoring Registration and Check

Goal

  • Register drift monitoring in the deployed service to monitor the trend of data.
  • You can schedule a monitoring cycle and receive a notification when an anomaly is detected in the data.

1. Monitoring Registration​

  • Go to the model screen in service.

  • Click the 모델 Drift(Model Drift) tab to move to the drift list screen.

  • Click the + 모니터링 등록(Monitoring Registration) button.

  • The registration form will pop-up then fill in the form as requirements *Note) Refer the instruction below the screen shot.

  • Click the 등록하기 button.

  • Monitoring Description: Write a description of the monitoring to be registered.

  • Base time: Enter the time to start the analysis. It must be after the present time.

  • Notification Configure: When an abnormality is detected in data, set the Slack channel to receive notifications.

  • Analysis cycle: Set the analysis cycle. It can be set in two ways: the default method and the cron expression.

    • Basic method: Set the analysis cycle in units of days/weeks/months from the base time.
    • Cron expression: Follow the guide to set the analysis cycle in Cron format.

  • Base data: Data analysis is performed by comparing two data sets. Determines the data set to compare with the new input data set.

    • Past data: Use the previous data set according to the analysis cycle for comparative analysis.
    • Training data: Use the data set used to train the model.

  • Target feature setting:

    • Type Specification: Specifies whether the data type of each feature is numeric or categorical.
    • Select Feature: Turn on/off the feature to be analyzed in the corresponding monitoring.
    • Threshold Setting: Set the threshold value for determining how big the change of each feature is as “anomaly data”. You can also set batches.

2. Monitoring Check​

  • After registration, move to the drift monitoring list page.

  • Drift analysis monitoring can be enabled/disabled.

  • As a result of the recent analysis, check whether an abnormality has been detected in “Drift or not”.

  • Select a Monitoring to go to the “Drift Results” page.

  • Monitoring Summary

    • Monitoring Status: Check whether the corresponding monitoring is active/inactive.
    • Last Run date: Check the last analysis date and time.
    • Next Run Date: Identifies the time when the next analysis will take place.
    • Whether Drift: Checks whether drift has been detected in the last run.

  • Top Drift Features: Check the top 3 features with the highest drift in the last analysis.

  • Drift Trend: View the drift trend of a feature graphically.

  • Drift history: View drift analysis history and drift scores.

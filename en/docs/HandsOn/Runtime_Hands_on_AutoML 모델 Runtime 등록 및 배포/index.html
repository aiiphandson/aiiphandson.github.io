1. AutoML Model Runtime Registration & Serving

Goals

  • Register and deploy the AutoML model to AI Runtime.
  • AutoML
    • Software to make ML model available for the non-expert. It enables to automate some or all ML modeling.
  • AI Runtime
    • Management service that enables you to register, validate, deploy the model in Accuinsight+
  • Validate model serving and review the history.
  • Register a new version of the model and Release Canary service.
  • Stop model serving.

1. Register new model​

  • Right-click the model node, and select 모델 레지스트리 등록 ** 새 모델 등록.

On 모델 레지스트리 등록 pop-up,

  • Enter the model name.
  • Select 중복확인 to check duplicate.
  • Select 확인. For OK
  • Select the model node. Registration time is provided under the 모델 레지스트리 항목 이력 section.
  • Select the button on the right-side of username. You’ll see the AI Runtime tab.
  • It is ready to deploy when the deployment status is changed to 배포가능(available).

2. Modify deployment information​

  • Select 서빙 시작 and select 배포자원 생성 to modify deploy resources.

(You’ll see the ‘서빙 설정’ tab, if you have resources already)

  • Modify deployment configuration on the “배포자원 생성” pop-up. 배포자원 설명(description) and 자원설정(resource) are required.
  • Select 생성완료.

ex 배포자원 설명 입력(required): “테스트 배포 자원” (Resource for test) à 자원설정(required): Medium -** GPU 설정 추가: off.

3. Model Serving​

  • Select 서빙 시작, and Go to 서빙 설정 tab.
  • Modify resources for serving, and Select 서빙 시작 on the bottom right to request for deployment.

ex Concurrency: 100(default) 자원설정(required): Small 선택 Replica: 1~1(default) GPU설정: off (default)

  • After a few minutes, the status change WAITING to RUNNING.

  • API test for prediction is available when the status is RUNNING.

4. Validate model serving​

  • Go to ‘예측이력’(Prediction History) tab, and Select 예측 테스트 to test.

  • The input data is determined automatically, and select 전송 to request a test.

  • Output data is returned on the right section.

  • Select 확인 and Select the refresh button right the 예측 테스트.

  • You’ll find the test history is added.

  • Click the history. You could review the test in detail.

5. Register model version​

  • Select ‘버전 현황’ tab.

  • Click the three vertical dots, and Select 워크플로우 조회 to go to the workflow of the model.

  • Right-click ‘AutoML’ node, and Select AutoML 설정하기.

  • On the ‘Leaderboard’ tab, select the 배포 button of another model which is not deployed.

  • Go to ‘workflow’.

  • You’ll see a new model node is added to the workflow.

  • Right-click the added model node, and Select 모델 레지스트리 등록 ** 새 버전으로 등록.

  • Select the model, and Select 확인.

  • Select the model node. Registration time is provided under the 모델 레지스트리 항목 이력 section.

  • Select the button on the right-side of username. Go to AI Runtime tab.

  • Check that the model status is 빌드중(building).

  • After a few minutes, the status changes to 배포가능(available to deploy).

6. Canary Release​

  • Select Canary 배포 추가 of the new model.

  • Modify the proportion of the Default service and Canary service

ex 서비스 버전(Default): 90%, 서비스 버전(Canary): 10%

  • Select the Canary service resource, and Click 서빙 시작.

  • The proportion and status of Canary service are provided on the section.

7. Model promotion​

  • Go to 서비스 대시보드(Service Dashboard) tab. Review the setting for serving.

  • Select 프로모션’(promotion) by clicking 상태 변경 of Canary service.

  • Select 확인 on the Canary 서빙 프로모션 pop-up.

  • The Default 서빙버전 is changed to the Canary serving version.

8. Stop Model Serving​

  • Select ‘중지’(stop) by clicking 상태변경 of Default 서빙버전.
  • Select ‘확인’ on the Default 서빙 중지(삭제) pop-up.
  • You'll see that the status is changed to DELETED.

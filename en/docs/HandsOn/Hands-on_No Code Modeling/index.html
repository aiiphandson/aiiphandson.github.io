6. AutoML Model Development

No Code Modeling

Goal

  • We will cover from creating new projects and workflows and from easily performing data preprocessing and model development using No Coding Tools such as 1) Wrangler and 2) AutoML.

    1) Wrangler : Spread sheet-based data pre-processing tool such as Excel for easy data processing
    2) AutoML: It is designed to easily develop machine learning models even by people without statistical knowledge or computer programming knowledge.

  • Generating prediction values by applying the new test data to AutoML model generated.

Login again with your ID and the new password

2. Creeat Workflow​

  • Click +워크플로우(+Workflow) button to create a new workflow

  • In the workflow type panel, select 빈 워크프로우(Empty Workflow) Empty Workflow and then click next button in the right bottom screen

  • In the Add Data Set screen, Click Asset button then the data source window pops-up
  • Select data source and then click the 등록(Registration) button.

    example select churn_train data

  • Return to the Add Data Set screen and click the Add button in the upper right corner.

3. Data Preprocessing​

  • (Preprocessing 1: Handling duplicate records) Click the Dataset node created in the workflow and click the Preprocess button on the right panel

  • Click Basic Processing > Drop Duplicate in the preprocessing menu.

  • In the Drop Duplicate setting screen on the right panel, click the +Add button to process duplicate values of the entire record

    note

    To process deduplicate values in a specific column, click the More button to designate the column name.

  • After the data process, the new data set will appear

  • (Preprocessing 2: Null value handling) Click the last Data Set node connected to the right of the Drop Duplicate node and click the Preprocess button in the right panel.

  • Click the Wrangler button in the preprocessing menu on the right panel.

    note

    Although preprocessing tasks can be processed in individual nodes (basic processing nodes, advanced processing nodes, etc.), many preprocessing tasks can be performed on one screen using Wrangler.

    Wrangler function (1) Cell-by-cell preprocessing: Remove spaces, find and replace, fill omissions, add strings (2) Column unit preprocessing: changing data type, sorting, dividing column, extracting string, creating New Column, Keep, Delete (3) Row preprocessing: Filter by value, filter top N%, Limit, deduplicate column, deduplicate all, remove missing column (4) Other preprocessing: Add Time Stamp, change date format, Time Move

  • Check the data on the Wrangler Grid screen, click the column name, and then right-click the column and select the preprocessing item from the menu window that appears.

  • To replace a known null value in a specific column, click Fill Na by col item in the preprocessing menu window

    Example Replace Null value in the Balance Column

  • In the right panel that appears later, enter a value to be replaced instead of the null value and click the Apply button.

    Example Replace Null value -> 0

  • (Preprocessing 3: Create a New Feature) To create a new column by calculating two columns, right-click on the BaseYear Column, choose withColumn in the preprocessing menu window.

    Example Create a new Tenure column by calculating the BaseYear column and the JoinYear column

  • Select column from selectType in the right panel that appears. Then, select two column names and an operator to be calculated, enter a new column name, and click the Apply button.

    Example col1 : BaseYear, Operator : (-), col2 : JoinYear, New Column Name : Tenure

  • When all pre-processing tasks are completed, click the Complete button in the upper right corner.

4. Data Exploration​

  • Double-click the Data Set node you want to explore in the workflow.

  • You can proceed with correlation analysis by clicking the “Correlation” tab on the data set screen.

  • Click the “Interactions” tab then you can proceed to explore the data distribution by chart and relationships between variables.

    Example > Analyzing the relationship between Y variable and X variable - Graph type: Column, X-axis: Exited, Y-axis: CreditScore, Calculate: Mean

note

Chart types :Line, Column, Bar, Pie, Scatter, Box Plot

  • When all data exploration tasks are completed, click the ‘<’ mark at the top left to exit the Dataset screen
  • You can also explore data by using the chart creation menu of the Data set node. Right-click the Data Set node you want to explore and click the “Create Chart” menu
note

Using the chart creation menu, you can use various types of charts than the data set > Interaction screen, and charts can be shared with other users because it has chart saving and report generation functions.

  • On the chart creation screen, click the button to the left of the X-axis display, and when the chart selection area appears, select a histogram chart among the chart types.

  • Select the column to be analyzed in the chart settings on the right panel and drag it to the column (base axis) column on the left.

    Example Select Age

  • Click the pencil icon that is displayed when you mouse over Untitled in the upper left corner to change the chart name

    Example Chart name: Age

  • Click the V icon to adjust the variable interval size by setting the interval value.

    Example value: 5 and click the “Save” button

  • Click the + button next to the chart name at the bottom left coner to create a new chart, or click the < mark at the top left to exit the chart creation screen when data exploration is complete.

  • Click “Create Report”

  • Drag Dataset > Age chart and add a Text box for a report

5. AutoML Learning > Setup and Startup​

note

AutoML Functions

  1. Data Preview
  2. Column check : Column name, data type, ratio of missing values, column distribution
    (distribution type,mode, eigenvalue, mean value, minimum value, maximum value)
  3. AutoML Setup
    a. Basic settings: analysis technique (classification/regression), target, and indicator selection
    b. Column unit setting: whether to use a column, change data type, replace missing values, remove outliers, select datatransformation method
    c. Data division: Holdout (Train/Valid/Test ratio selection), CV (number of folds, Test ratio selection)
    d. Sampling: Select sampling target dataset (all data or train data), input sample size, select balancing function (Oversampling / Undersampling)
    e. advanced settings : Timeout Setup (All Timeout Setup, Algorithm unit timeout setting), whether to use ensemble model, Estimator select (Select which ML algorithm to use), One Hot Encoding for categorical variables (Level Encoding applied when not selected ), Scaling Method for Numeric Variables (Standardize, MinMax, Normalize),
    Feature Engineering : Feature select (SelectPercentile, SelectKBest, SelectFwe, Boruta), Feature dimension reduction (PCA, FeatureAgglomeration), Feature augmentation (Polynomial
  4. EDA : Correlation Matrix, Pair Plot, Variable Analysis, Parallel Coordinates
  5. Leader Board : Model name, model ranking, model performance (Metric), model information (Pipeline), model chart (Feature Importance, Confusion Matrix, AUC, Lift, etc.) selection, final model selection (provisioning)

For AutoML training, click the final Data set node and click the Modeling button on the right panel.

  • Select AutoML from the right panel modeling type.

  • Click the +Add button in the shape right panel.

  • AutoML Double-click the node or right-click the mouse and select Configure AutoML.

  • On the AutoML screen, check the analysis method, target variable, and indicator set in the basic setting, and change the default setting value if necessary.

    Example Analysis method: Classification, Target: Exited, Indicator: F1

  • Click the Data Columns Tab in the upper left to check column information and, if necessary, perform column-by-column setting

    Example Use checkbox: Exclude CustomerId, Surname, and BaseYear columns

  • If data segmentation is required, select Holdout or CV method from the Segmentation option.

    Example Holdout > Use Test > Train : Valid : Test Ratio = 6 : 2 : 2

  • If sampling is required, select Sampling Target data from the Sampling option and enter the Sampling Size. And if Oversampling or Undersampling is required, click the [Balancer] button.

  • If you double-click, the bar graph of the standard class in the pop-up window that appears the size of the comparison class is adjusted to be the same as the size of the standard class.

    Ex) If you double-click ‘0’ Class, the size of ‘1’ Class is adjusted to be the same as ‘0’ Class Balancer

  • If other additional settings are required, goto the advanced settings option and proceed with additional settings for algorithms and Feature Engineering.

  • When all settings are completed, click the Run button in the upper right corner.

  • When a pop-up window to re-check the settings appears, click the Execute button after confirming the settings.

6. AutoML Learning > Leaderboard​

  • When AutoML execution is completed, a model node, which is the best one, is created in the workflow, and if you click Model Node,

  • You can check the model performance on the right panel. And to check the detailed model result, double-click AutoML Node.

  • Click the Leaderboard Tab to check the training model results, such as algorithm ranking and model performance by algorithm. If you want to change the final model, click the Deploy button on the right side of the algorithm, and when all tasks are completed, click the ‘<’ mark in the upper left to exit the screen.

  • To change the name of a workflow, click the Pencil icon that is displayed when you mouse over the name of the workflow in the upper left corner, change the name and click the V icon.

  • Click the Save button at the top right to save the workflow. < Example > Workflow name : AutoML Learning

7. AutoML Prediction​

  • Click Model Node and then click the Generate Workflow for Prediction button in the right panel

  • When the “Guide to Restriction of Replicating Workflow for Prediction” pop-up window appears, Click the Apply button

    note

    Among the record processing functions used for learning purposes, “Drop Duplicates” processing and sampling functions are excluded from prediction purposes. :::

  • Go back to the Dashboard > Projects screen and double-click on the created workflow for prediction with the name Predict.~ in the workflow name.

    Example Predict.AutoML Learning

  • Enter the workflow for prediction and click “Data Import” in the “Get Data” node.

  • Click the Asset button on the Add Dataset screen, when the Asset Selection window pops up, select the data for prediction and click the Register button.

    Example select “churn_test”

  • Click the +Add button at the top right of the Add Dataset screen

  • Enter the workflow, click Model Node and then click the Predict button in the upper right corner.

  • After the prediction data set node is properly connected next to the Model node, double-click the prediction data set node and check the prediction data generated

  • Go to the last column in the prediction data set and check the Predict_Value value, which is the predicted value column. Then, click the dotted area at the top right corner to save the data.

    Example Save as csv file by selecting download

  • The predicted data can be sent to data storage and to databases

  • Right-Click the predicted data set node then click Data Export

  • Select a target data storage or DB (S3, RDB, HDFS, Blob, Snowflake)

  • After the workflow creation is completed, it can be scheduled for batch jobs.
